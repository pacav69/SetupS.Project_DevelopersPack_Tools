Install & Regen modes        


SetupS operates in two modes. They are self-determined by SetupS automatically but if one is aware of which mode it could be operating in, the behavior of SetupS becomes more predictable (and much easier to understand). For example, <Assembly> is ignored during Regen-mode.

  • Install-mode

This is the mode whenever an application or game is being installed for the first-time or for updates. Generally it is activated whenever SetupS processes the application or game from the source-path, which is typically the path of the deployment package from whatever medium it resides on; or the system temp folder if the deployment packages is an archive file (such as .apz or .pgz).


  • Regen-mode

So called because it effectively "regenerates" the application or game without actually installing it. And it does so by recreating shortcuts, processing registry entries, executing scripts, etc. This mode is activated whenever SetupS processes the application or game from its Installed-to path.
